Angular HMR Lazy Components

Out-of-the-box Angular HMR reloads lazy routes, but this example shows how it can be used to reload lazy (dynamically-loaded) components.

Instructions:

  • Hit F12 to open the Chrome Dev Tools. Open the Console, so you can see HMR messages.
  • Click on a mixture of the "Add" buttons below. A "loading" graphic simulates the component loading its data.
  • Click on any open component to close it.
  • With a mixture of components open, make a change to the source code in a.component.ts. Only the instances of component A will be hot-reloaded. Component A is in one module, and Components B1 and B2 are in a different module.
  • Try something similar with b1.component.ts - all B1 and B2s will refresh, because they're part of the same module.
